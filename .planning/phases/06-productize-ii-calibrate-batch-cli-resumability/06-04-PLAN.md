# Plan 06-04: Batch Generation Module

## Objective

Wave 3: create `src/musicgen/batch.py` with `generate_batch(config) -> BatchResult` using `ProcessPoolExecutor` (spawn context), per-sample resume logic, failure isolation, and structured JSON progress logs to stderr. Populate `tests/test_batch.py` with fast tests (monkeypatched `generate`).

This plan depends on 06-02 (Config has `output_mode` and `count`) but does NOT depend on 06-03 (calibrate.py). `generate_batch` wraps `generate` which already handles pre-roll gracefully.

## Tasks

### Task 1: Create src/musicgen/batch.py

**File changes:**
- `src/musicgen/batch.py` (new) â€” implement verbatim per `06-PATTERNS.md` Pattern 1.

Key implementation points:
1. Module-level imports: `json`, `logging`, `multiprocessing`, `os`, `sys`, `time`, `concurrent.futures`, `dataclasses`, `datetime`, `typing`. Also `from config import Config`, `from musicgen.api import SampleResult, generate`, `from musicgen.manifest import ManifestWriter`, `from musicgen.seeds import derive_sample_seed`.
2. `_NullManifestWriter` class with `append(self, entry)` no-op and delegating `is_sample_complete` static method.
3. Module-level `_worker(global_seed, sample_index, config)` function â€” MUST be at module level (picklable). Creates per-sample `Config` with overrides and calls `generate(worker_config, manifest_writer=_NullManifestWriter())`.
4. `BatchResult` frozen dataclass with `slots=True`: `total`, `succeeded`, `failed`, `skipped`, `results: Tuple[SampleResult, ...]`, `duration_seconds`.
5. `generate_batch(config)` function:
   - `max_workers = config.workers or os.cpu_count() or 1`
   - `mp_ctx = multiprocessing.get_context("spawn")`
   - Emit `batch_start` event.
   - Create `ManifestWriter(config.dataset_root)` in main process.
   - For each `sample_index in range(config.count)`:
     - If `ManifestWriter.is_sample_complete(config.dataset_root, sample_index)` â†’ increment `skipped`, log, continue.
     - Else: emit `sample_start`, submit `_worker` to pool.
   - In `as_completed` loop: get result, call `_append_manifest`, emit `sample_done`, increment counters, collect `results`.
   - Handle `future.result()` exception: build synthetic failed `SampleResult`, append manifest, emit `sample_done` with `status="failed"`.
   - Return `BatchResult`.
6. `_append_manifest(manifest_writer, result, config)` helper.
7. `_log_event(event, **kwargs)` helper â€” writes JSON line to `sys.stderr`.
8. Zero bare `random.*` (trivially satisfied â€” no randomness in batch.py; generate() owns RNG).

**Important:** `Config.__dict__` works for reconstruction ONLY if `Config` is a regular dataclass (non-frozen mutable). If `Config` is frozen, use `dataclasses.asdict(config)` or `dataclasses.replace()`. Check: `config.py` uses `@dataclass` (NOT frozen). `Config.__dict__` is available. Use `dataclasses.replace(config, global_seed=global_seed, sample_index=sample_index)` for clarity and correctness (handles fields that don't appear in `__dict__` due to slots).

```python

    config, global_seed=global_seed, sample_index=sample_index

```

### Task 2: Populate tests/test_batch.py

**File changes:**
- `tests/test_batch.py` â€” remove `pytest.mark.skip`; implement 10 fast tests.

All tests monkeypatch `musicgen.batch._worker` OR `musicgen.api.generate` to avoid real FluidSynth. Pattern: use a fake `generate` that returns a `SampleResult(status="ok", ...)` immediately.

```python
@pytest.fixture
def fake_generate(monkeypatch, tmp_path):
    """Replace generate() in batch._worker with a fast stub."""
    call_count = [0]

    def _fake_generate(config, *, manifest_writer=None):
        call_count[0] += 1
        idx = config.sample_index
        seed = 0
        sample_dir = os.path.join(config.dataset_root, f"{idx:06d}")
        os.makedirs(os.path.join(sample_dir, "stems"), exist_ok=True)
        os.makedirs(os.path.join(sample_dir, "midi"), exist_ok=True)
        # Write sentinel
        sentinel = os.path.join(sample_dir, "sample.json")
        import json
        with open(sentinel, "w") as f:
            json.dump({"sample_index": idx, "split": "train",
                       "duration_seconds": 1.0, "musicality_score": 0.5}, f)
        return SampleResult(
            sample_index=idx, seed=seed, sample_dir=sample_dir,
            sample_json_path=sentinel,
            mix_path=os.path.join(sample_dir, "mix.wav"),
            stem_paths={}, midi_paths={},
            split="train", status="ok",
            musicality_score=0.5, duration_seconds=1.0,
        )

    monkeypatch.setattr("musicgen.batch._worker.__wrapped__", _fake_generate)
    # Alternative: monkeypatch generate in the module where batch imports it.
    monkeypatch.setattr("musicgen.api.generate", _fake_generate)
    return call_count
```

Tests:
```
TestBatchResult:
  test_batch_result_fields â€” BatchResult has total/succeeded/failed/skipped/results/duration_seconds
  test_batch_result_frozen â€” cannot set attribute on BatchResult

TestGenerateBatch:
  test_generate_batch_count_zero â€” config.count=0 â†’ BatchResult.total=0, succeeded=0
  test_generate_batch_count_one â€” config.count=1 â†’ one generate() call, BatchResult.total=1, succeeded=1
  test_generate_batch_skips_complete â€” pre-create sample.json for index 0 â†’ skipped=1
  test_generate_batch_retries_failed â€” pre-create manifest entry with status=failed but no sample.json â†’ NOT skipped (retried)
  test_generate_batch_failure_isolation â€” fake_generate raises for index 1 of 3 â†’ succeeded=2, failed=1, batch continues
  test_generate_batch_manifest_appended â€” after batch, manifest.jsonl has count lines

TestProgressLog:
  test_batch_start_event_emitted â€” capture stderr; verify "batch_start" event JSON present
  test_batch_done_event_emitted â€” capture stderr; verify "batch_done" event with correct counts
```

Note: for `test_generate_batch_failure_isolation`, the fake `generate` raises an exception for a specific sample_index. The batch should catch this via `future.result()` exception handling and produce a failed SampleResult.

Note: testing with `ProcessPoolExecutor` and spawn context in unit tests is slow. Use `max_workers=1` and rely on the spawn context initialization to be fast. Alternatively, test the orchestration logic by patching `concurrent.futures.ProcessPoolExecutor` with a `ThreadPoolExecutor` in tests (faster, same API). Decision: use `monkeypatch.setattr("musicgen.batch.ProcessPoolExecutor", ThreadPoolExecutor)` for test speed. This does not test spawn vs. fork but does test the orchestration logic. Document this as a test constraint.
