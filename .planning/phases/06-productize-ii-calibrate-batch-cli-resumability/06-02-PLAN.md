### Task 2: Extend writer.write_sample with output_mode + pre_roll_offset_s

**File changes:**
- `src/musicgen/writer.py` â€” add two keyword-only parameters to `write_sample`:
  - `output_mode: str = "full"` (D-66)
  - `pre_roll_offset_s: float = 0.0` (D-53)

  Conditional write logic (D-66):
  ```python
  _WRITE_MIX   = output_mode in ("full", "mix-only")
  _WRITE_STEMS = output_mode in ("full", "stems-only")
  _WRITE_MIDI  = output_mode in ("full", "midi-only")
  ```

  Beat-time shift (D-53) â€” new helper `_apply_preroll_offset(anno_copy, offset_s)`:
  ```python
  def _apply_preroll_offset(anno_copy: dict, offset_s: float) -> dict:
      if offset_s == 0.0:
          return anno_copy
      for key in ("beat_times", "downbeat_times"):
          if key in anno_copy and isinstance(anno_copy[key], list):
              anno_copy[key] = [
                  round(t - offset_s, 6)
                  for t in anno_copy[key]
                  if t - offset_s >= 0.0
              ]
      return anno_copy
  ```
  Called after `copy.deepcopy(annotation)` and before `_rewrite_paths_relative`.

  Sum-of-stems assertion guard: only call `_assert_sum_of_stems` when `_WRITE_STEMS and _WRITE_MIX`.

  The returned paths dict keys are conditional: `stems_*` keys are absent if `not _WRITE_STEMS`; `midi_*` keys absent if `not _WRITE_MIDI`; `mix` absent if `not _WRITE_MIX`.

  In `sample.json`, the `stems` dict is `{}` when `not _WRITE_STEMS`; the `midi` dict is `{}` when `not _WRITE_MIDI`; the `mix` field is `""` when `not _WRITE_MIX`. The `pre_roll_offset_seconds` annotation field is populated from the passed `pre_roll_offset_s` (annotation already has this key; writer overwrites it via the deep-copy route).
