**Then implement `src/musicgen/writer.py`** (~200-250 lines). Required surface (from Plan 05-04 and RESEARCH §Code Examples):

- `def write_sample(dataset_root, sample_index, annotation, mix_working_path, stems_working_paths, midi_working_paths, *, seed, musicgen_version, split, config) -> SampleResult` — atomic sentinel write (midi → stems → mix → sum-of-stems → json.rename), returns dataclass.
- `def _concat_layer_stems(stem_paths_per_part, layer, out_path) -> str` — per-layer pydub concat across arrangement parts (mirror `mixer.concat_parts`).
- `def _concat_layer_midis(midi_paths_per_part, render_results, tempo, time_signature, layer, out_path) -> str` — **the subtle one**: absolute-tick walk (RESEARCH §Code Examples has ready-to-use helper). Do NOT naively concat tracks — midiutil note tracks have `sum(msg.time) < part_duration_ticks`, which places next-part notes too early. Walk to absolute ticks with `offset_ticks = mido.second2tick(render_result.duration_seconds, ticks_per_beat, tempo_us)`, merge, emit back as delta ticks.
- `def _assert_sum_of_stems(mix_path, stem_paths, epsilon) -> Tuple[bool, float]` — **int32 accumulator** (four int16 stems summed overflow int16; must cast to int32 before summing). Normalize to float32 via `/ 32768.0`. Compare `np.max(np.abs(diff)) < epsilon`.
- `def _rewrite_paths_relative(annotation: dict) -> dict` — deep-copy annotation, rewrite `mix` to `"mix.wav"`, `stems.*` to `"stems/{layer}.wav"`, `midi.*` to `"midi/{layer}.mid"` (D-11 per-sample-dir-relative).
- `sample.json` serialized with `json.dump(..., sort_keys=True, indent=2, separators=(",", ": "))` (D-23 — load-bearing for Plan 05-06 D-30 cross-check hash stability).
- Use `os.rename(tmp, final)` for the sentinel `sample.json` — POSIX-atomic within same filesystem.
- `shutil.rmtree(working_dir)` after sentinel lands (unless `config.keep_working_dirs`).
